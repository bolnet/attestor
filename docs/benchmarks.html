Methodology

We evaluate Memwright against two industry-standard memory benchmarks. Both are run via our built-in CLI with no manual intervention — fully automated ingestion, retrieval, and scoring.

LOCOMO

Long Conversation Memory

Tests memory systems on realistic multi-session conversations between two people. Evaluates single-hop recall, multi-hop reasoning, temporal understanding, and open-domain questions.

Dataset 10 conversations, 20-30 sessions each
Questions 1,540 across 4 categories
Scoring Accuracy (LLM-judge) + Token F1
Source arXiv:2402.17753

MAB

MemoryAgentBench · ICLR 2026

Comprehensive memory benchmark covering accurate retrieval (AR), conflict resolution (CR), temporal ordering (TTL), and long response understanding (LRU).

Dataset 146 examples across 4 categories
Questions 60-200 per example
Scoring Exact match + Substring match + Token F1
Source arXiv:2501.13953

Infrastructure & Models

Every component of the test environment is documented for full reproducibility.

Component Technology Details
Embedding Model OpenAI text-embedding-3-small 1536 dimensions, via OpenRouter API
LLM (Answer Synthesis) Llama 3.3 70B Versatile Via Groq API — free, open-source model
LLM (Judge) Llama 3.3 70B Versatile Same model for LOCOMO accuracy scoring
Storage Backend SQLite (local mode) Brute-force cosine similarity + BFS graph traversal
Retrieval Pipeline 3-layer cascade + RRF Tag Match → Graph Expansion → Vector Similarity → RRF Fusion
Hardware Apple Silicon (local) M-series Mac, 16GB+ RAM
Runtime Python 3.14 memwright v0.1.3
Note on LLM usage: The LLM is used only for answer synthesis (generating responses from retrieved context) and accuracy judging. The retrieval pipeline itself uses no LLM re-ranking — only tag matching, graph traversal, vector search, and RRF fusion.

Reproduce These Results

Every benchmark can be reproduced with a single CLI command.

Setup
# Install with benchmark dependencies
pip install memwright[benchmark]

# Set API keys
export OPENROUTER_API_KEY="your-key"   # for embeddings
export GROQ_API_KEY="your-key"         # for LLM (free)
LOCOMO Benchmark
# Run all 10 conversations
agent-memory locomo \
  -v \
  -o locomo-results.json

# Or run specific conversations in parallel
agent-memory locomo \
  --skip-conversations 4 \
  --max-conversations 2 \
  -v -o locomo-5-6.json
MAB Benchmark
# Run all 4 categories
agent-memory mab \
  --categories AR CR TTL LRU \
  -v \
  -o mab-results.json

# Or run categories separately
agent-memory mab \
  --categories AR \
  -v -o mab-ar.json

Execution Details

How the benchmarks were actually run, including parallelization strategy and rate limit handling.

Parallelization 7 parallel processes — MAB categories + LOCOMO conversation slices run independently
Rate Limit Handling Exponential backoff retry (8 attempts, max 120s wait) for API 429 errors
Embedding Provider OpenRouter → OpenAI text-embedding-3-small (auto-detected)
LLM Provider Groq API (llama-3.3-70b-versatile, OpenAI-compatible endpoint)
Total Runtime ~4.5 hours total (API-bound, not compute-bound)
Cost ~$2 (OpenRouter embeddings) + $0 (Groq LLM) = ~$2 total
