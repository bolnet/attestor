  • Black-box memory you can't inspect or debug

The Solution

pip install and go. Everything auto-provisions.

Memwright runs entirely on your machine. SQLite, ChromaDB, and NetworkX auto-provision on first use. Your memory is a directory you can inspect, query, and version control.

  • Zero config. No Docker. No containers.
  • Local embeddings via sentence-transformers
  • SQLite + JSON files you can inspect with raw SQL
  • Automatic contradiction resolution
  • Precise recall within configurable token budgets

RRF Scoring (k=60)

score = Σ 1/(k + rank). Memories appearing in multiple layers get additive boosts. Single-source queries just deduplicate.

Post-Processing

Temporal boost (+0.2 max, decays over 90 days). Entity boost (+0.15 to +0.30). Budget fitting: greedy token selection, always at least 1 result.

The Stack

Three stores. Zero infrastructure.

Everything runs in-process. No servers, no containers, no network calls.

🗃

SQLite

Core storage with ACID guarantees. Always available. Run raw SQL to inspect your memories. Built into Python.

🔭

ChromaDB

In-process vector store with sentence-transformers embeddings (all-MiniLM-L6-v2). No API keys. Persistent on disk.

🔗

NetworkX

In-memory entity graph with multi-hop BFS traversal. JSON persistence. Sub-millisecond queries.

📈

RRF Fusion

Reciprocal Rank Fusion combines all three layers. Memories appearing across layers get boosted. Temporal and entity boosts applied.

Design Principles

Opinionated by design.

Zero-Config

AgentMemory("./path") creates directory, schema, collections, and config. No setup step. Everything auto-provisions on first use.

Non-Fatal Failure

ChromaDB and NetworkX are wrapped in try/except. If they fail, the system continues with SQLite only. Graceful degradation, not crashes.

Immutable Supersession

Memory content is never mutated. New facts supersede old ones via linked list. Full history preserved. "Works at Google" replaces "Works at Meta" cleanly.

Eager Persistence

SQLite commits after every write. NetworkX saves graph.json after every entity add. No batching means no data loss on crash.
