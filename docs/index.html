The Problem

AI agents forget everything between conversations. The typical fix is a managed vector database or cloud memory service with high latency and vendor lock-in.

The Solution

Memwright gives agents an open, inspectable memory store you can query with SQL. pgvector for semantic similarity, Neo4j for entity graph traversal, and automatic contradiction detection — fused together with Reciprocal Rank Fusion.

Features

🔍

Open & Inspectable

Your memories live in a SQLite file. Run sqlite3 memory.db and see exactly what the agent knows. No black boxes, no vendor lock-in.

🎯

Token Efficient

300-500 tokens per recall vs 15,000+ for full history replay. Precise context injection within a configurable token budget.

🔄

Contradiction Handling

New facts automatically supersede old ones. "Works at Google" replaces "Works at SoFi" — no stale memories.

🔬

Graph-Enhanced

Neo4j entity graph enables multi-hop reasoning. Query "Python" and find related "FastAPI" memories through graph traversal.

🔌

MCP Server

First-class Claude Code and Cursor integration via Model Context Protocol. 7 memory tools out of the box.

⚡

No Re-ranking LLM

Retrieval uses tag matching, graph traversal, and vector search with RRF fusion. No expensive LLM judge or re-ranker at recall time.

Quick Start

Claude Code / Cursor (MCP)
# Initialize a memory store
agent-memory init ~/.agent-memory/my-project

# Add to your .mcp.json
{
  "agent-memory": {
    "command": "agent-memory",
    "args": ["serve", "~/.agent-memory/my-project"]
  }
}
Python API
from agent_memory import AgentMemory

mem = AgentMemory("./my-agent")

# Store facts
mem.add("User prefers Python over Java",
        tags=["preference"], category="preference")

# Recall relevant memories
results = mem.recall("what language?")
context = mem.recall_as_context("user background")
