Our Mission

We hate dementia. We can't cure it in humans.
But we will cure it in AI.

Every AI agent built today has the same disease. It forgets. Not gradually. Not partially. Completely. Every session. Every time.

Four hours of debugging a database migration, thirty back-and-forth messages about schema evolution, a breakthrough at 11pm — and then you close the terminal. Tomorrow morning, your agent has no idea any of it happened.

This isn't a feature gap. It's a fundamental failure of how agents are built.

We solve one problem: agent memory.

Not retrieval. Not orchestration. Not prompt engineering. Memory. The ability to store what matters, forget what doesn't, find what's relevant, and never pollute the context window with noise.

Under the Hood

Three retrieval layers. Six scoring signals.
Zero LLM calls.

Vector search alone finds what's similar. Not what's relevant. Not what's recent. Not what's correct. We run three layers in parallel, score with PageRank + confidence decay + MMR diversity, and deduplicate by content hash. Every query. Every time.

🏷️

Layer 1: Tag Match

SQLite

Exact and partial tag hits. Fast. Deterministic. The foundation of every query.

🕸️

Layer 2: Graph Expansion

NetworkX

Multi-hop entity traversal. Query "Python" and find "FastAPI," "Django," "pip" through graph edges.

🔮

Layer 3: Vector Search

ChromaDB

Semantic similarity for when exact matches miss. Catches meaning, not just keywords.

⚡

Fusion + Scoring

RRF or Graph-Aware Blend · PageRank · Confidence Decay

RRF fuses multi-layer results. PageRank boosts well-connected entities. Confidence decays over time (-0.001/hr) and boosts on access (+0.03). Switch to graph-aware blend for entity-heavy workloads.

🎯

MMR Diversity + Budget Fitting

Maximal Marginal Relevance · Token Budget Control

MMR eliminates near-duplicate results via Jaccard similarity. Then budget fitting packs the top memories into your 2K token window. The other 9,996 memories never enter context.

Multi-Agent

Namespace isolation. RBAC. Provenance.
Built for orchestrated pipelines.

Every agent gets its own memory partition. Orchestrators control budgets. Researchers get read-only access. Every write is traced back to the agent that made it.

# Multi-agent pipeline with provenance + RBAC from agent_memory.context import AgentContext, AgentRole ctx = AgentContext.from_env( agent_id="orchestrator", namespace="project:acme", role=AgentRole.ORCHESTRATOR, token_budget=20000, ) # Spawn isolated child contexts (immutable — returns new) planner = ctx.as_agent("planner", role=AgentRole.PLANNER, token_budget=5000) researcher = ctx.as_agent("researcher", role=AgentRole.RESEARCHER, read_only=True) # Provenance auto-enriched on every write planner.add_memory("Architecture: event sourcing", category="technical") # Recall is namespace-scoped + cached within session results = researcher.recall("architecture decisions")

Namespace Isolation

Each agent, project, or team gets its own memory partition. No cross-contamination.

6 RBAC Roles

ORCHESTRATOR, PLANNER, EXECUTOR, RESEARCHER, REVIEWER, MONITOR. Read-only mode. Write quotas.

Full Provenance

Agent trail, parent tracking, session IDs, visibility levels. Every write traced to its source.

Compliance Built In

Review flags, compliance tags, session summaries for audit. SOC2-friendly by design.

Python API

Not just MCP. A full Python library.

Import it. Use it. No server required. Same retrieval pipeline, same scoring, same contradiction handling — directly in your Python code.

Store & Recall
from agent_memory import AgentMemory mem = AgentMemory("./my-agent") # auto-provisions everything mem.add("User prefers Python over Java", tags=["preference"], category="preference") results = mem.recall("what language?", budget=2000) context = mem.recall_as_context("user background", budget=4000)
Contradiction Handling
# Automatic supersession mem.add("User works at Google", category="career") mem.add("User works at Meta", category="career") # ^ Google memory auto-superseded # Maintenance mem.forget(memory_id) # Archive mem.forget_before("2025-01-01") # Archive old mem.export_json("backup.json") # Export mem.import_json("backup.json") # Import (dedup)

What We Believe

Opinionated. On purpose.

Zero config beats configuration.

AgentMemory("./path") creates everything. No init. No YAML. No wizard.

Degradation beats failure.

If ChromaDB crashes, SQLite continues. If NetworkX fails, tags + vectors continue. You never lose memory because an optional layer broke.

History beats deletion.

New facts supersede old ones. They never overwrite. Full audit trail. Always.

Math beats LLMs in the retrieval path.

Every LLM call adds cost, latency, and non-determinism. We rank with algorithms. Same query, same results. Every time.

Dedup beats bloat.

SHA-256 content hashing on every add() and import. Same content never stored twice. Your memory stays lean without manual cleanup.

Your disk beats their cloud.

sqlite3 ~/.memwright/memory.db "SELECT * FROM memories LIMIT 10;" — see what your agent knows. Not an API call. The actual data.
