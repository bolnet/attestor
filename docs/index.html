The Problem

AI agents forget everything between conversations. The typical fix is a cloud memory service with high latency, vendor lock-in, and zero visibility into what your agent actually remembers.

The Solution

Memwright runs entirely on your machine. One docker compose up gives you pgvector + Neo4j + SQLite — the same stack that scores 83.3% on LOCOMO. Your agent's memory is a file you can inspect, query, and version control.

Local Dev to Production in 3 Steps

Build and test your agentic AI locally with Claude Code. When it works, deploy the same codebase. No migration, no API changes, no surprises.

1

Start Locally

2 minutes

Docker Compose spins up PostgreSQL (pgvector) and Neo4j on your machine. One pip install. Your agent has semantic search, entity graphs, and contradiction detection — all running at localhost.

docker compose up -d
pip install memwright[all]
agent-memory init ~/.agent-memory/my-project
agent-memory doctor ~/.agent-memory/my-project
2

Build with Claude Code

5 minutes

Add the MCP server to your Claude Code config. Now every conversation builds memory automatically — entities extracted, embeddings computed, contradictions resolved. Your agent remembers preferences, decisions, and context across sessions.

// .mcp.json — that's it
{
  "agent-memory": {
    "command": "agent-memory",
    "args": ["serve", "~/.agent-memory/my-project"]
  }
}
3

Ship to Production

Same codebase

Point the connection strings at your production PostgreSQL and Neo4j. The Python API is the same, the retrieval pipeline is the same, your memories carry over. No vendor migration — you already own the data.

# Same code, production infra
from agent_memory import AgentMemory

mem = AgentMemory("./prod-store",
  pg_conn="postgresql://prod-host:5432/memwright",
  neo4j_uri="bolt://prod-neo4j:7687")

# 300-500 tokens per recall, 83.3% LOCOMO accuracy
context = mem.recall_as_context("user preferences")

Why Local-First Matters for Agentic AI

🔎

Debug What Your Agent Knows

Run sqlite3 memory.db and see every memory, every entity, every relationship. When your agent gives a wrong answer, you can trace it back to a specific stored memory — not a black-box API response.

⚡

Iterate Without API Costs

Test memory extraction, retrieval, and contradiction handling locally. Only the embedding step needs an API call ($0.002 per recall). No metered retrieval fees, no per-query charges, no usage surprises.

🔒

Your Data Stays on Your Machine

Memories live in a SQLite file on disk. Entity graphs stay in your local Neo4j. Nothing is sent to a cloud memory service. Ideal for enterprise, regulated industries, or anyone who cares about data sovereignty.

🚀

Production Path Without Migration

Your local Docker setup mirrors production exactly. Same pgvector, same Neo4j, same retrieval pipeline. When you deploy, change connection strings — not architecture. Agents built locally work identically in production.

Quick Start

Claude Code / Cursor (MCP)
# Initialize a memory store
agent-memory init ~/.agent-memory/my-project

# Add to your .mcp.json
{
  "agent-memory": {
    "command": "agent-memory",
    "args": ["serve", "~/.agent-memory/my-project"]
  }
}
Python API
from agent_memory import AgentMemory

mem = AgentMemory("./my-agent")

# Store facts
mem.add("User prefers Python over Java",
        tags=["preference"], category="preference")

# Recall relevant memories
results = mem.recall("what language?")
context = mem.recall_as_context("user background")
