The Problem

AI agents forget everything between conversations. The typical fix is a cloud memory service with high latency, vendor lock-in, and zero visibility into what your agent actually remembers.

The Solution

Memwright runs entirely on your machine. Tag matching, entity graph traversal, and semantic vector search — all fused with Reciprocal Rank Fusion. Your agent's memory is a file you can inspect, query, and version control.

From Zero to Memory in 2 Steps

One Prompt

Paste this into Claude Code. It handles the install, Docker containers, memory store initialization, and MCP config. You get tag matching, graph traversal, vector search, embeddings, and contradiction detection.

# Paste this into Claude Code:

Set up memwright as this project's persistent memory.

1. Install: pip install "memwright[all]"
   (if pip fails on macOS, use: pipx install "memwright[all]")
2. Make sure Docker Desktop is running
3. Initialize: agent-memory init ~/.agent-memory/PROJECT_NAME
4. Add embedding API key to the .env file
5. Create .mcp.json with the memory server config
   (include env vars for OPENROUTER_API_KEY,
   PG_CONNECTION_STRING, NEO4J_URI, NEO4J_PASSWORD)
6. Add memory instructions to CLAUDE.md
7. Verify: agent-memory doctor ~/.agent-memory/PROJECT_NAME

Why Local Matters for Agentic AI

🔎

Debug What Your Agent Knows

Run sqlite3 memory.db and see every memory, every entity, every relationship. When your agent gives a wrong answer, you can trace it back to a specific stored memory — not a black-box API response.

⚡

Iterate Without API Costs

Test memory extraction, retrieval, and contradiction handling locally. Only the embedding step needs an API call ($0.002 per recall). No metered retrieval fees, no per-query charges, no usage surprises.

🔒

Your Data Stays on Your Machine

Memories live in SQLite on disk. Entity graphs stay in your local Neo4j. Vector embeddings stay in your local pgvector. Nothing is sent to a cloud memory service.

🚀

Same Stack, Any Scale

Your local setup uses the same pgvector, same Neo4j, same retrieval pipeline as any deployed version. Change connection strings to scale — not architecture.

Manual Setup
# 1. Install
pip install "memwright[all]"

# 2. Initialize (starts pgvector + Neo4j via Docker)
agent-memory init ~/.agent-memory/my-project

# 3. Add your API key
echo 'OPENROUTER_API_KEY=sk-or-...' >> \
  ~/.agent-memory/my-project/.env

# 4. Add to .mcp.json
{
  "mcpServers": {
    "memory": {
      "command": "agent-memory",
      "args": ["serve", "~/.agent-memory/my-project"],
      "env": {
        "OPENROUTER_API_KEY": "sk-or-...",
        "PG_CONNECTION_STRING": "postgresql://...",
        "NEO4J_URI": "bolt://localhost:7687",
        "NEO4J_PASSWORD": "memwright"
      }
    }
  }
}
Python API
from agent_memory import AgentMemory

mem = AgentMemory("./my-agent")

# Store facts
mem.add("User prefers Python over Java",
        tags=["preference"], category="preference")

# Recall relevant memories
results = mem.recall("what language?")
context = mem.recall_as_context("user background")
