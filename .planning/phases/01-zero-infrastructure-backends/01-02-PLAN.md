  <action>
    Create `agent_memory/store/chroma_store.py` with class `ChromaStore`:

    **IMPORTANT DESIGN DECISION**: Use ChromaDB as an in-process persistent client (via `chromadb.PersistentClient`), NOT via MCP stdio subprocess. The CONTEXT.md mentions MCP stdio, but ChromaDB's PersistentClient is simpler, more reliable, and still zero-config. The MCP stdio approach was based on claude-mem's pattern but adds unnecessary complexity for our use case. ChromaDB handles embeddings internally with sentence-transformers.

    Constructor: `__init__(self, store_path: Path)`
    - Create `chromadb.PersistentClient(path=str(store_path / "chroma"))`
    - Get or create collection named "memories" with `embedding_function` using ChromaDB's built-in `SentenceTransformerEmbeddingFunction(model_name="all-MiniLM-L6-v2")`
    - This means NO separate embedding step is needed — ChromaDB handles it

    Methods (duck-type compatible with old VectorStore + orchestrator expectations):
    - `add(self, memory_id: str, content: str) -> None` — upsert document into collection with id=memory_id, documents=[content]. No need to pass embedding — ChromaDB generates it from the document text.
    - `search(self, query_text: str, limit: int = 20) -> List[Dict[str, Any]]` — query collection with query_texts=[query_text], n_results=limit. Return list of dicts: `{"memory_id": id, "content": document, "distance": distance}`. ChromaDB returns distances (lower = more similar).
    - `delete(self, memory_id: str) -> bool` — delete from collection by id. Return True if existed.
    - `count(self) -> int` — return collection.count()
    - `close(self) -> None` — no-op (PersistentClient handles cleanup)

    NOTE: The old VectorStore.add() took (memory_id, content, embedding). The new ChromaStore.add() takes (memory_id, content) — no embedding parameter because ChromaDB generates embeddings internally. This signature change will be handled in Plan 03 when wiring into core.py.

    NOTE: The old VectorStore.search() took a query_embedding vector. The new ChromaStore.search() takes query_text string — ChromaDB embeds the query internally. The orchestrator will need a small change in Plan 03 to pass text instead of embedding.

    Write tests in `tests/test_chroma_store.py` using a tmp_path fixture for isolation.
  </action>

    Rewrite `agent_memory/graph/networkx_graph.py` to replace Neo4j with NetworkX:

    Class `NetworkXGraph`:

    Constructor: `__init__(self, store_path: Path)`
    - `self._path = store_path / "graph.json"`
    - `self._graph = nx.MultiDiGraph()`
    - If `self._path` exists, call `self.load()`

    Node convention: Node id = `name.lower()`. Store attributes: `display_name` (original case), `entity_type`, plus any extra attributes.

    Methods (matching Neo4jGraph interface exactly):
    - `add_entity(self, name: str, entity_type: str = "general", attributes: Optional[Dict] = None) -> None`
      - Key = name.lower(). MERGE behavior: if node exists, update attributes (do not overwrite entity_type if already set, unless new one is more specific). Always set display_name.
    - `add_relation(self, from_entity: str, to_entity: str, relation_type: str = "related_to", metadata: Optional[Dict] = None) -> None`
      - Normalize relation_type same as old code (_sanitize_rel_type — uppercase, safe chars).
      - Auto-create from/to nodes if they don't exist (type="general").
      - Add edge with key=relation_type, plus metadata attributes.
    - `get_related(self, entity: str, depth: int = 2) -> List[str]`
      - BFS/DFS from entity.lower() up to `depth` hops in BOTH directions (undirected traversal on directed graph).
      - Return display_name of found nodes (not the queried node itself).
      - Return empty list if entity not found.
    - `get_subgraph(self, entity: str, depth: int = 2) -> Dict[str, Any]`
      - Return `{"entity": entity, "nodes": [...], "edges": [...]}` matching Neo4jGraph format.
      - nodes: list of `{"name": display_name, "type": entity_type, "key": node_id}`
      - edges: list of `{"source": from_key, "target": to_key, "type": relation_type}`
    - `get_entities(self, entity_type: Optional[str] = None) -> List[Dict[str, Any]]`
      - Return all nodes as dicts. Filter by entity_type if provided.
    - `get_edges(self, entity: str) -> List[Dict[str, Any]]`
      - Return edges involving entity as `{"subject": from_display, "predicate": rel_type, "object": to_display, "event_date": metadata.get("event_date", "")}`.
      - Used by orchestrator for graph_context_triples.
    - `stats(self) -> Dict[str, Any]`
      - Return `{"nodes": n, "edges": e, "types": {type: count}}`
    - `save(self, path: Optional[str] = None) -> None`
      - Serialize graph to JSON using `nx.node_link_data(self._graph)` and write to self._path (or provided path).
    - `load(self, path: Optional[str] = None) -> None`
      - Read JSON from self._path (or provided path) and reconstruct graph via `nx.node_link_graph(data)`.
    - `close(self) -> None`
      - Call `self.save()`.

    Save timing: save after every write operation (add_entity, add_relation). This is simple and safe for the expected data volume (hundreds to low thousands of nodes). Per user decision: Claude's discretion on save timing.
