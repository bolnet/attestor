    **core.py __init__():**
    Replace the stubbed vector_store and graph initialization with:
    ```python
    # Initialize ChromaDB vector store
    from agent_memory.store.chroma_store import ChromaStore
    self._vector_store = ChromaStore(self.path)

    # Initialize NetworkX graph
    from agent_memory.graph.networkx_graph import NetworkXGraph
    self._graph = NetworkXGraph(self.path)
    ```
    No try/except wrapping for import — these are required dependencies now (installed via pip). Wrap the constructor calls in try/except to catch runtime errors (e.g., disk full) but log and continue — SQLite path should never break.

    **core.py add():**
    Replace the commented-out vector store block with:
    ```python
    if self._vector_store:
        try:
            self._vector_store.add(memory.id, content)
        except Exception:
            pass  # Non-fatal
    ```
    Note: NO embedding import needed — ChromaStore handles embeddings internally.

    Keep the graph block exactly as before but remove the old `from agent_memory.embeddings import get_embedding` reference. The graph block (extract_entities_and_relations + add_entity + add_relation + save) should work unchanged since NetworkXGraph has the same interface.

    **core.py search():**
    Update the vector search path. The old code called `get_embedding(query)` then `self._vector_store.search(query_embedding)`. Replace with:
    ```python
    if query and self._vector_store:
        try:
            vec_results = self._vector_store.search(query, limit=limit * 2)
            # ... rest of filter logic stays the same
        except Exception:
            pass
    ```
    No embedding import needed — ChromaStore.search takes text.

    **core.py batch_embed():**
    Rewrite to iterate all active memories and call `self._vector_store.add(mem.id, mem.content)` for each. ChromaDB handles embedding internally, so batch_embed just ensures all memories are indexed. Return count of memories processed.

    **core.py close():**
    Update to call `self._vector_store.close()` and `self._graph.close()` (which calls save).

    **core.py health():**
    Complete rewrite. Check three components:
    1. **SQLite** — keep existing check (query count, db_path, size, latency)
    2. **ChromaDB** — `self._vector_store.count()`, check that chroma/ dir exists
    3. **NetworkX Graph** — `self._graph.stats()`, check that graph.json exists (or is new/empty)
    4. **Retrieval Pipeline** — list active layers (always all 3 now)

    No Docker checks. No container inspections. No embedding API checks. Simple and fast.

    **retrieval/orchestrator.py:**
    Update Layer 3 (vector similarity). The old code imported `get_embedding`, computed query embedding, then called `self.vector_store.search(query_embedding)`. Replace with:
    ```python
    if self.vector_store:
        try:
            vec_results = self.vector_store.search(query, limit=20)
            for vr in vec_results:
                memory = self.store.get(vr["memory_id"])
                if memory and memory.status == "active":
                    distance = vr.get("distance", 1.0)
                    score = max(0.0, 1.0 - distance)
                    results.append(
                        RetrievalResult(memory=memory, score=score, match_source="vector")
                    )
        except Exception:
            pass
    ```
    Remove the `from agent_memory.embeddings import get_embedding` import entirely from orchestrator.py.
