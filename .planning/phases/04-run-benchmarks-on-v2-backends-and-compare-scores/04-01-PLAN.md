  <files>benchmark-logs/mab-v2-results.json</files>
  <action>
1. Check OPENROUTER_API_KEY is set: `echo $OPENROUTER_API_KEY | head -c 10` (should show prefix, not empty). If missing, STOP and report — benchmarks cannot run without it.

2. Ensure benchmark dependencies are installed: `.venv/bin/pip install datasets openai` (MAB needs HuggingFace datasets, both need openai client).

3. Run MAB sanity check (quick, ~2 min): `.venv/bin/python -m agent_memory.cli mab --max-examples 2 --verbose`. If this fails, diagnose and fix before proceeding. Common issues: import errors (backends changed), ChromaDB initialization, missing deps.

4. Run full MAB benchmark: `.venv/bin/python -m agent_memory.cli mab --verbose`. This runs AR + CR categories by default. Expect 10-30 minutes. Capture stdout.

5. Save the full output to benchmark-logs/mab-v2-results.json. If the CLI prints JSON directly, save that. If it prints human-readable output, capture the key scores (AR%, CR%, Overall%) into a JSON file:
```json
{
  "benchmark": "MAB",
  "version": "v2",
  "date": "2026-03-15",
  "backends": "ChromaDB + NetworkX",
  "scores": {
    "AR": <number>,
    "CR": <number>,
    "overall": <number>
  },
  "raw_output": "<full stdout>"
}
```

Note: MAB uses OPENROUTER_API_KEY for answer synthesis (gpt-4.1-mini via OpenRouter). The scoring itself is deterministic (token F1, exact match).
